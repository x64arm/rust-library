## 4. 问题记录与解决方案

### 4.1 PostgreSQL 18 编译问题

#### 问题 1：ICU 库缺失

**错误信息：**
```
configure: error: Package requirements (icu-uc icu-i18n) were not met
```

**解决方案：**
```bash
dnf install -y libicu-devel
```

#### 问题 2：数据库权限问题

**错误信息：**
```
Failed to run migrations: ExecuteMigration(Database(PgDatabaseError { 
  severity: Error, code: "42501", message: "must be owner of table books"
}))
```

**原因分析：** SQLx 迁移需要表的所有者权限

**解决方案：**
```sql
-- 授权表所有权
ALTER TABLE books OWNER TO library_admin;
ALTER TABLE readers OWNER TO library_admin;
ALTER TABLE borrow_records OWNER TO library_admin;
ALTER TABLE users OWNER TO library_admin;

-- 或重新创建数据库（推荐）
TRUNCATE books, readers, borrow_records, users RESTART IDENTITY CASCADE;
```

#### 问题 3：重复数据导致迁移失败

**错误信息：**
```
duplicate key value violates unique constraint "readers_email_key"
```

**解决方案：**
```sql
-- 清空数据后重新迁移
TRUNCATE books, readers, borrow_records, users RESTART IDENTITY CASCADE;
```

### 4.2 后端编译问题

#### 问题 4：Axum 0.8 路由语法变更

**错误信息：**
```
expected struct `axum::routing::Route`, found enum `Result`
```

**原因分析：** Axum 0.8 版本路由参数语法从 `:id` 改为 `{id}`

**解决方案：**
```rust
// 旧语法（错误）
.route("/api/books/:id", get(handlers::get_book))

// 新语法（正确）
.route("/api/books/{id}", get(handlers::get_book))
```

### 4.3 前端开发问题

#### 问题 5：编辑和删除按钮无响应

**原因分析：** 缺少 onClick 事件处理函数

**解决方案：**
```tsx
// 添加事件处理
const handleEdit = (record: Book) => {
  setEditingBook(record);
  editForm.setFieldsValue(record);
  setIsEditModalOpen(true);
};

<Button onClick={() => handleEdit(record)}>编辑</Button>
```

#### 问题 6：新增数据后列表不刷新

**原因分析：** 缺少数据重新加载逻辑

**解决方案：**

#### 问题 7：界面布局混乱

**问题描述：** 侧边栏和内容区域缩到左上角

### 4.4 部署问题

#### 问题 9：外网无法访问

**原因分析：** 腾讯云安全组未开放 80 端口

**解决方案：**
1. 登录腾讯云控制台
2. 进入安全组配置
3. 添加入站规则：端口 80，协议 TCP，来源 0.0.0.0/0

#### 问题 10：Nginx 配置冲突

**错误信息：**
```
nginx: [emerg] bind() to 0.0.0.0:80 failed (98: Address already in use)
```

**原因分析：** 默认 server 块占用了 80 端口

**解决方案：**
```nginx
# 修改默认配置
server {
    listen 80 default_server;
    server_name jp.x64arm.com _;
    # ...
}
```

### 5.2 部署步骤

#### 5.2.1 编译前端

```bash
cd frontend
npm run build
cp -r dist/* /opt/library-system/www/
```

#### 5.2.2 编译后端

```bash
cd backend
cargo build --release
cp target/release/library_backend /opt/library-system/bin/
```

#### 5.2.3 配置 systemd 服务

**/etc/systemd/system/library-backend.service**

```ini
[Unit]
Description=Library Management System Backend
After=network.target postgresql.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/library-system/bin
EnvironmentFile=/opt/library-system/bin/.env
ExecStart=/opt/library-system/bin/library_backend
Restart=on-failure

[Install]
WantedBy=multi-user.target
```

#### 5.2.4 Nginx 配置

**/etc/nginx/conf.d/library-system.conf**

```nginx
server {
    listen 80 default_server;
    server_name jp.x64arm.com;

    location / {
        root /opt/library-system/www;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location /health {
        proxy_pass http://127.0.0.1:3000;
    }
}
```
